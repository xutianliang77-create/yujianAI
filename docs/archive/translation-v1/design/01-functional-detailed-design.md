# 语见AI功能详细设计（翻译方向历史归档）

> 已于 2026-07-17 被“中国 LiveKit 类实时平台”目标取代，不再作为现行设计。

版本：v1.0  
日期：2026-07-17  
状态：设计评审稿，尚未开发

## 1. 产品目标

语见AI是以实时翻译为主线、以语音 Agent 和会后智能为扩展的 AI 通讯平台。用户
不需要理解 ASR、MT、TTS、LiveKit 或模型部署，只需选择沟通方式、语言、声音和
是否允许 AI 协助。

产品承诺：

- 原始转写、忠实翻译、AI 建议和会后摘要分层展示，不相互冒充。
- 网络或模型降级时优先保留原文字幕，再降为半双工，最后才结束。
- Agent 代表用户说话、拨号或调用工具前必须有明确授权。
- 录音、声音克隆、转写和高风险工具分别同意，可撤销、可审计。

## 2. 用户和角色

| 角色 | 主要目标 |
| --- | --- |
| 注册用户 | 发起面对面、聆听、Call Link、翻译电话和 Agent 任务 |
| Guest | 无需安装 App，通过链接加入翻译通话 |
| PSTN 对端 | 通过普通电话参与翻译或 Agent 通话 |
| 企业成员 | 使用组织词库、知识库、策略和共享额度 |
| 运营/客服 | 查看脱敏质量、处理退款和用户申诉 |
| 安全/审计员 | 审核授权、工具、录音、异常和删除记录 |

LiveKit participant 是媒体身份，不等于产品角色。角色、授权和计费由 Control API
维护。

## 3. 产品信息架构

移动端一级入口：

1. **沟通**：面对面、聆听、Call Link、翻译电话。
2. **AI 助理**：Agent Assist、AI 外呼、任务记录。
3. **记录**：字幕、译文、摘要、重点、待办和导出。
4. **工具**：Type-to-Speak、扫描翻译、术语和声音。
5. **我的**：账号、订阅、隐私、模型位置、设备和诊断。

Web Guest 只包含入房、授权、媒体控制、字幕、译音和离开，不暴露账号管理和内部
模型配置。

## 4. 能力优先级

| 优先级 | 能力 |
| --- | --- |
| P0 | 账号/同意、面对面、聆听、Call Link、字幕、忠实翻译、TTS、历史、用量 |
| P1 | 全双工抢话、术语、说话人、Type-to-Speak、会后摘要、Agent Assist |
| P2 | PSTN 翻译、AI 外呼、DTMF、接管、warm transfer、声音身份、Egress |
| P3 | 扫描翻译、会议增强、企业工作流、RTMP/WHIP、跨应用扩展 |

## 5. 通用会话模型

所有实时模式都创建一个 `communicationSessionId`。会话通用状态：

```text
created -> admitting -> active -> ending -> ended
    |          |          |          |
    +----------+----------+----------+-> failed
```

通用界面元素：

- 会话模式和网络/模型位置。
- 源语言、目标语言和自动识别状态。
- 原文、译文、speaker、置信提示和 revision 标记。
- 开始、暂停、继续、静音、停止朗读、结束。
- 当前降级状态和可恢复动作。
- 计时、剩余额度和隐私状态。

终态后不允许恢复同一个媒体会话；“再次开始”创建新
`communicationSessionId`，历史可关联前一会话。

## 6. 面对面双向翻译

### 6.1 用户流程

1. 选择双方语言或开启自动识别。
2. 选择处理位置：自动、端侧优先、云端优先。
3. 选择字幕-only 或字幕+朗读。
4. 点击开始并完成麦克风/云端处理同意。
5. 双方轮流讲话；系统显示原文、译文和当前说话人。
6. 用户可点击单段重播、纠错反馈或复制。
7. 结束时 flush 尾句，保存历史并结算。

### 6.2 行为规则

- 自动语言识别按 turn 决策，不把整个 session 永久锁死。
- 中英混合、姓名、数字、地址、型号和缩写不触发错误反向翻译。
- 字幕先显示，TTS 不阻塞下一段 ASR/翻译。
- 扬声器模式启用回声 gate；耳机模式允许更连续的全双工。
- 降级顺序：完整双工 → 半双工 → 字幕-only → 端侧基础能力。

### 6.3 异常

- 麦克风拒绝：保留设置页和权限修复入口，不创建 active session。
- 云端超时：显示原文并尝试 fallback；不得用错误提示覆盖原文。
- TTS 失败：保留字幕并允许稍后重试单段播放。
- 结束中断：本地立即进入 ending，后台 outbox 继续完成保存和结算。

## 7. 聆听翻译

适用于会议、课堂、演讲和采访。

- 默认单向采集、连续大字幕、关闭自动朗读。
- 支持固定源语言或自动识别，允许多个目标语言显示。
- 支持说话人匿名标签、重命名和低置信度标记。
- 支持锁屏/后台策略，但必须明确平台限制和采集状态。
- 长时间会话按章节和静音间隔分段，防止一个无限增长的转写对象。
- 用户可标记重点；AI 摘要在会后异步生成，不影响实时字幕。

## 8. Call Link翻译通话

### 8.1 Host

1. 创建链接，选择语言、有效期、最大人数、TTS 和是否录音。
2. 分享一次性 URL；页面显示等待状态和 Guest 同意情况。
3. Guest 入房后开始双向字幕和定向译音。
4. Host 可静音、移除 Guest、关闭译音或结束会话。

### 8.2 Guest

1. 打开链接，查看 AI 翻译、转写、录音和数据处理说明。
2. 输入会话内显示名并授权麦克风。
3. 用一次性 ticket 换取短期 LiveKit token。
4. 加入后只看到当前通话所需控制。
5. 离开或被移除后旧 ticket 不可再次使用。

### 8.3 媒体和字幕

- Host、Guest 和 Runtime 具有不同最小权限 token。
- 人类 participant 只能发布麦克风，默认不能发布 data、视频或屏幕。
- 字幕/状态只接受可信 Runtime 或服务器 topic。
- 每个方向拥有独立 TTS target leg 和 playback generation。
- Guest 短暂断线可重连媒体，但可靠历史由 API 快照补齐。

## 9. PSTN翻译电话

### 9.1 发起

- 用户选择号码、主叫显示号码、语言、声音、最大时长和费用上限。
- API 在拨号前校验国家/地区、禁拨号码、余额、并发和录音合规。
- 创建 usage hold 后才创建 SIP participant。
- 接通前显示 ringing/early media，不提前进入 billable active。

### 9.2 通话

- App 用户和 PSTN 对端各有独立 participant/leg。
- 双方原始音频分别进入翻译链路，译音定向回到另一条 leg。
- DTMF 作为显式用户操作或受控 Agent 工具，不写入普通字幕。
- 不支持 clear 的电话 provider 强制半双工，禁止伪报全双工。

### 9.3 结束和对账

- 业务结束、SIP 状态、provider webhook 和计费结算必须幂等。
- 忙线、拒接、无人接、无路由和 provider 错误映射为结构化原因。
- 未接通不按通话时长收费；外部 provider 固定成本单独记录。

## 10. Agent Assist

人类保持通话控制，Agent 默认不自动发声。

功能：

- 建议回复和一键朗读。
- 实时重点、姓名、号码、地址、金额和待办卡片。
- 术语/知识库检索结果。
- 风险提示和追问建议。
- 一键请求接管、转人工或保存任务。

AI 建议必须标记为建议，用户确认后才生成目标语言译音。Agent 不得更改原始字幕和
忠实译文。

## 11. Autonomous Agent

### 11.1 任务创建

用户配置：

- 目标号码或联系人。
- 明确任务目标和成功条件。
- 可接受时间、预算和重试次数。
- 允许使用的工具和风险上限。
- 用户监听、暂停、取消和接管方式。

系统生成计划预览、风险说明和费用估计；用户授权后创建 `agentTaskId`。

### 11.2 执行

状态：

```text
draft -> authorized -> queued -> dispatching -> dialing
      -> disclosure -> active -> completed/failed/cancelled
```

要求：

- 开场披露 AI 身份及录音/转写状态。
- L0 只读工具可自动执行；L1 可撤销动作需任务授权；L2 二次确认；L3 禁止自动执行。
- 工具失败不得伪造成功。
- 用户接管时立即停止新工具和 TTS，用户 leg 成为主动讲话方。
- 结果包含成功条件、证据、未解决项和下一步。

## 12. Warm transfer

1. 原 caller 进入 hold。
2. 创建隔离 consultation room。
3. Transfer Agent 呼叫人工目标并提供脱敏摘要。
4. 人工接受后移动/桥接 participant。
5. Agent 介绍双方后离开。
6. 无人接或拒绝时回到 caller，说明结果并提供替代方案。

转接是多阶段业务流程，不以 LiveKit participant 移动成功单独判定业务成功。

## 13. Type-to-Speak

- 用户输入或粘贴文字，选择源/目标语言和目标 participant。
- 系统保护号码、地址、字母串、金额和日期。
- 显示原文、译文和实际播报文本三层预览。
- 用户确认后生成 TTS，可插入当前会话记录。
- Agent 场景中 Type-to-Speak 仍属于用户明确发言，不算 Agent 自主输出。

## 14. 字幕、记录和会后智能

### 14.1 实时

- partial 可覆盖当前临时行，不写入可靠历史。
- final 创建 segment；revision 只能单调增加。
- 原始 ASR、规范化文本、忠实译文和 AI 结果分别保存。

### 14.2 会后

- 基本信息：标题、模式、时间、时长、语言、参与者。
- 四个视图：摘要、重点/待办、双语全文、质量与诊断。
- 摘要和事实必须能回溯到 segment evidence。
- 用户可重命名 speaker、修正文案和导出；修正不覆盖原始证据。

### 14.3 删除

- 删除正文、摘要和媒体对象。
- 账本只保留法律/财务所需的脱敏引用。
- 删除任务异步执行时显示状态和失败重试。

## 15. 声音与身份

- 预置音色由服务端能力目录决定，客户端不展示不可用音色。
- 个人声音录入需独立同意、质量检查、试听和撤销。
- 参考音频和 embedding 使用独立对象权限。
- 声音身份识别只在明确授权的 session 启用；低置信度回退匿名 speaker。
- 撤回后新会话不得继续匹配，历史按用户删除策略处理。

## 16. 扫描翻译

P3 功能，使用独立 OCR/版面 Provider：

- 拍照或导入图片。
- OCR block 保留坐标、方向和置信度。
- 按 block 翻译并支持原图/译图切换。
- 没有可靠坐标时降级为逐块列表，不伪造版面回贴。
- 默认不将图片送入实时通信 session；保存时创建独立 artifact。

## 17. 订阅、额度和计费

- 创建付费 session 前预留额度。
- active 时间和 provider usage 分开记录。
- append-only ledger，禁止直接修改余额结果。
- 重复 End、webhook 和恢复不得重复结算。
- 余额不足时提前提示；耗尽后先停止新翻译/TTS，再安全结束。
- 退款、赠送和人工调整具有独立原因、操作者和审计。

## 18. 设置和隐私

- 语言、声音、自动播放、术语、处理位置、保存期限。
- 清楚显示端侧、私有云或公共云能力。
- 录音、转写、声音克隆、诊断上传分别开关。
- 导出诊断默认脱敏，不包含字幕正文、token 和号码明文。
- 开发/实验模型只在独立 model-lab 出现，不进入生产 App 设置。
